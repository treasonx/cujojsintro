Curljs





What is it?

A fast AMD and resource loader.

What is it like?

RequireJS 

What problem does it solve

Makes it simple to breakup large application into smaller more manageable pieces. 

The pieces can be individually loaded during development but also

example


define(['jquery'], function($) {
  return function(ele) {
    $(ele).text('oh hai!');
  }
});

curl(['myModule'], function(myModule) {
  myModule();
});

Whenjs





What is it?


Fast Promise/A+ implementation.

What is it like?


The Q promise library.

What problem does it solve?


Simple management of async code. 

Sane error management for async code.

example


var def = when.defer();
var promise = def.promise;
promise.then(function() {
  console.log('All Done');
}, function() {
  console.log('Error!!!');
});

meldjs





What is it?


AOP for JavaScript

What is it like?


Nothing that I know of for JS.

What Problem does it solve?


Add behavior to objects.

No Need to modify the source function.

Aspects can be shared across Objects.

Keep your code simple and DRY.

example


meld.around(obj, 'getUsers', function(joinpoint) {
  var def = when.defer();
  var promise = def.promise;
  var client = joinpoint.args[0];
  client.query('BEGIN');
  console.log('Starting Transaction');
  promise.then(function() {
    client.query('COMMIT');
  }, function() {
    client.query('ROLLBACK');
  });
  promise.always(client.release);
  joinpoint.args.unshift(def);
  joinpoint.proceed();
  return promise;
});

msgsjs





What is it?

PubSub and Message Buses.

What is it like?

Lots of pubsub out there, nothing this full featured.

What problem does it solve?


Manage the complexity of message passing

Easily send messages to/from client to server.

Add adaptors to transform and handle messages.

example


var bus = require('msgs').bus();
bus.channel('tweet');
bus.outboundAdapter(function(tweet) {
  console.log(tweet.text);
}, {input: 'tweet'});

wirejs





What is it?


Application Composition (I.O.C.)

What is it like?


Nothing I know of in the JS world.

What problem does it solve?


Easily "wire up" dependencies in your application based on config!

curl(['hello/app/controller',
'text!hello/app/template.html',
'i18n!hello/app/strings.js',
'hello/app/view', 
'jquery'], 
function(Controller, template, strings, View, $) {
  var $ele = $('span:eq(0)'); 
  var controller = new Controller($ele);
  var view = new View();
  controller.on(view);
  view.render(template, strings);
});


define({ // Wire spec
  controller: {
    create: 'hello/app/controller',
    properties: {node: { $ref: 'first!span', at: 'view' }},
    on: { view: { 'input': 'update' } }
  },
  view: {
    render: {
      template: { module: 'text!hello/app/template.html' },
      replace: { module: 'i18n!hello/app/strings.js' }
    },
    insert: { last: 'root' }
  },
  $plugins: ['wire/dom', 'wire/dom/render', 'wire/on']
});
